Maintaining a Multilingual CV in LaTeX

Lately I had to update my CV and stumbled on a little old problem.

Being italian mothertongue I had two different version of my CV, one in italian and one in english.

This made tedious making edits, mainly because each time that I’ve added a line on the file I realized there were other places where an update was needed.

Remove a line here, move this section there, repeat for the other file.

Errr….What I’ve moved where?

My solution, while pretty hacky, is to maintain only one file with both translations paired, line by line; for achieving this I’ve written a simple command, defined as:

% switch language from lang 1 to lang 2 changing the number after #
\newcommand{\lan}[2]{#1}

A complete example of a document in two languages would be:

\documentclass{article}
% switch language from lang 1 to lang 2 changing the number after #
\newcommand{\lan}[2]{#1}

\begin{document}
  \lan{Hello World!}{Salve, mondo!}
\end{document}

The \lan command defined on line 3 gets two parameters and outputs the first; in this case, the document’s first language is english.

The output will be just a document containting “Hello World!”.

If you change the \lan command and instead of #1 you put #2, you’ll magically get the line “Salve, mondo!”, the italian version of hello world.

Ok, this is not perfect: every time you want to change language you must edit the command, and also the compilation won’t automatically generate different pdf based on the language.

But at least you’ll have only one file to edit.

P.s.

If instead of [2] we define the command with [3] we’ll be able to obtain a document in three languages, as long as the command is used with three pairs of braces instead of two (one per language).

We can increment the number of languages as much as we want, but the code readability will drop.
